Когда для сохранения этого файла vs code стал запрашивать рутовые права, нашел такое решение: cd to your project folder and run sudo chmod -R 777 <project_dir_name> and done
Я разобрался! VS code запрашивает сохранение от рута для этого файла, потому что сам сохраняет файлы по умолчанию как сторонний пользователь, не имуя рутовых прав. Когда же я дал права всем пользователям (sudo chmod 666 .travis.yml) записывать этот файл, то все стало сохраняться.
